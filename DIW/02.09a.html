Formularios

Los formularios HTML son herramientas que permiten a los usuarios introducir datos que pueden ser enviados a un servidor para su procesamiento. Consisten en diferentes tipos de controles como campos de texto, botones, casillas de verificación, listas desplegables, etc.

Funcionamiento básico:

  • El usuario completa los campos del formulario
  • Al enviar el formulario, los datos se recopilan
  • Los datos se envían al servidor mediante HTTP (GET o POST)
  • El servidor procesa la información y puede devolver una respuesta
  • Los formularios son esenciales para interactuar con aplicaciones web

GET/POST

Método GET

Envía los datos del formulario a través de la URL como parámetros de consulta.

Ventajas:

  • Los datos son visibles en la URL
  • Puede ser marcado como favorito
  • Útil para búsquedas y filtros
  • Más rápido para datos pequeños

Método POST

Envía los datos en el cuerpo de la petición HTTP, no en la URL.

Ventajas:

  • Los datos no son visibles en la URL
  • Puede enviar grandes cantidades de datos
  • Más seguro para información sensible
  • Puede enviar archivos

Atributos

accept-charset
Especifica los caracteres que el servidor puede procesar para los datos del formulario.
autocomplete
Indica si el navegador puede completar automáticamente los valores del formulario.
enctype
Especifica cómo se codifican los datos del formulario al enviarlos al servidor.
novalidate
Indica que el formulario no debe ser validado al enviarse.
target
Especifica dónde mostrar la respuesta recibida después de enviar el formulario.

Atributos button

  1. Formaction - Especifica dónde enviar los datos del formulario
  2. formmethod - Define el método HTTP para enviar datos
  3. formenctype - Especifica cómo codificar los datos del formulario
  4. formnovalidate - Indica que no se valide el formulario al enviar
  5. formtarget - Especifica dónde mostrar la respuesta del formulario
